F3 total layout attrs: 36 (8 colcount + 28 colspan + 0 wrappers)

data-skin is identical (17 each) — skins are variant-independent.

Analysis

F3 wins on: Zero wrappers (less DOM). Slightly fewer total attributes for uniform grids. Single-column content is free (CSS default spans). Shallower DOM (max depth 3).

F4 wins on: Explicit row boundaries (no span-counting to find rows). Single-column sections need zero layout attributes at all — no data-colcount on section. Fewer child colspan attrs in mixed layouts (checklist: 6 vs 9) because auto-span-1 elements in wrappers don't need the attr. Clearer separation: layout structure (article) vs content (section children).

The real tradeoff: F3 trades implicit row boundaries for fewer DOM nodes. F4 trades extra article wrappers for explicit structure. In patterns with many uniform cells (stats grid), both have the same colspan cost — F4 just adds wrappers. In mixed patterns (checklist, title bar), F4 saves colspan attrs because auto-spanning elements inside a wrapper don't need explicit colspan. The deeper question: is an LLM more likely to get wrong — implicit row boundaries (span accumulation) or consistent wrapper usage?
